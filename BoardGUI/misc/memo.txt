Database
> BoardDB

Table
> 유저		(users)			유저 권한 : admin(최고 관리자), manager(부운영자), user(기본)
> 게시판		(boards)		다중 게시판 제작을 위한 게시판 설정
> 게시글		(posts)			메인 데이터, 게시판 내 유저가 작성하는 글
> 댓글		(comments)		게시글 종속 데이터, 작성된 글의 대한 답글
> 첨부파일	(attachments)	게시글 종속 데이터, 작성된 글의 대한 첨부파일

Table_users
> 고유 id			user_id
> 로그인 id			username		(30) 중복불가
> 비밀번호			password		(255) 해싱으로 인해 길이가 길어지므로 여유롭게
> 표시용 닉네임		nickname		(30) 중복불가
> 생일				birth_data		date, 년도/월/일
> 전화번호			phone			(20)
> 이메일				email			(100) 중복불가
> 권한				role			권한(admin(최고 관리자), manager(부운영자), user(기본))
> 계정 활성 여부		is_active 		기본값 true(활성화)
> 가입일				created_at		time stamp, 생성시간 기준

Table_boards
> 고유 id			board_id
> 게시판 코드			code			(30) 중복불가, not null
> 게시판 이름			name			(50) not null
> 게시판 타입			type			NOTICE(공지사항), NORMAL(기본값), QNA(건의사항) 등
> 읽기 권한			read_role		int default 1, 권한(admin(최고 관리자, 9), manager(부운영자, 5), user(기본, 1))에 따라 사용
> 쓰기 권한			write_role		int default 1, 권한(admin(최고 관리자, 9), manager(부운영자, 5), user(기본, 1))에 따라 사용
> 게시판 활성여부		is_active		기본값 true(활성화)

Table_posts
> 고유 id			post_id
> 어느 게시판글인지		board_id		(FK)
> 누가 작성하였는지		user_id			(FK)
> 제목				title			(200), not null
> 내용				content			mediumtext, not null
> 조회수				view_count		int default 0
> 작성일				created_at		timestamp
> 수정일				updated_at		timestamp
> ------ 위에까지는 공통사항 -------
> 상단 고정 여부		is_notice		default false
> 비밀 글 여부			is_secret		default false
> 처리 상태			status			(20) default waiting

Table_comments
> 고유 id			comment_id	
> 어느글의 댓글인지		post_id			(FK), not null
> 누가 작성하였는지		user_id			(FK), not null
> 공개 여부			is_secret		default false
> 삭제 여부			is_deleted		boolean default false > 화면 표시용
> 내용				content			(1000), not null
> 작성일				created_at		timestamp

Table_attachments
> 고유 id			file_id
> 어느 글의 파일인지	post_id			(FK), not null
> 유저가 올린 파일명	origin_name		(255), not null
> 서버에 저장된 파일명	save_name		(255), not null
> 저장 경로			save_path		(255), not null
> 파일 크기			file_size		not null
> 확장자명			ext				(10)
> 업로드 시간			created_at		timestamp




2. GUI
> 로그인
입력칸 : 아이디, 비밀번호
버튼 : 로그인, 취소, 회원가입, 종료 (이벤트 생성)

> 회원가입
입력칸 : 아이디, 비밀번호, 닉네임, 생년월일, 전화번호, 이메일
버튼 : 등록, 취소, 돌아가기 (이벤트 생성)

> 메인화면
회원정보, 관리자용, 게시판(공지사항, 자유게시판), 게시판 추가
버튼 : 메인이동, 내 정보, 로그아웃, 종료

> 내 정보 확인
현재 로그인 중인 회원 정보 표시
닉네임, 생년월일, 전화번호, 이메일, 가입날짜 표시
회원 탈퇴 기능 추가

> 내 정보 수정
내 정보 확인 화면과 비슷하게 구성하지만 수정할 수 있도록

> 내 정보 수정_비밀번호 변경
현재 로그인 중인 회원 정보에서 비밀번호 재입력 후 바꾸고 싶은 비밀번호 작성
